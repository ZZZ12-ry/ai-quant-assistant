# 简单有效的均线量化策略

这是一套中长线趋势跟踪策略。它想解决的问题不是“均线能不能判断趋势”，而是“均线在震荡行情中反复缠绕时，如何少被假信号打扰”。策略先把均线形态变得更稳定，再用 OBV 确认成交量动能，最后等价格突破 N 周期高低点才入场，并用随持仓时间收紧的动态移动止损退出。

一句话理解：**先确认趋势真的顺，再确认成交量支持，最后等价格突破；进场后用越来越紧的移动止损保护利润。**

## 交易流程图

```text
原始K线
  |
  v
计算 VWMA1 / VWMA2 / VWMA3 和 OBV
  |
  v
锁定均线：震荡时尽量压平均线噪声
  |
  v
趋势过滤：价格在 VWMA3 有利一侧，MAOBV 同向变化
  |
  v
形态过滤：价格位置、均线排列、均线斜率同时顺向
  |
  v
突破触发：突破 N 周期高点做多，跌破 N 周期低点做空
  |
  v
持仓管理：跟踪有利极值，K 从 1 递减到 0.3，止损逐步收紧
  |
  v
触及动态止损线后平仓
```

## 虚拟K线理解图

下面不是回测数据，而是帮助理解策略的简化示意。

```text
价格
  ^
  |                         ▲ 入场：价格突破 N 周期高点
  |                    ┌─────┐
  |              ┌─────┘     │
  |        ┌─────┘           │      橙色线：动态止损，持仓越久越贴近价格
  |   ┌────┘              ···················
  |───┘────────────────────────────────────────> 时间
      VWMA1  >  VWMA2  >  VWMA3
      MAOBV 上升
      锁定均线未被震荡噪声频繁拉动
```

研究时看这张图要关注三件事：

- 入场不是因为“均线上穿”一个条件，而是趋势、形态、动能、突破共同满足。
- 动态止损不是固定止盈，它会跟随开仓后的有利价格移动。
- 如果 K 衰减太快，止损会很快贴近价格，可能保护利润，也可能让趋势还没展开就提前离场。

## 核心概念解释

| 概念 | 含义 | 研究时要看什么 |
| :--- | :--- | :--- |
| VWMA | 成交量加权移动平均线，成交量越大的K线权重越高。 | 趋势是否由有成交量的价格推动，而不是轻成交噪声。 |
| 三均线结构 | VWMA1、VWMA2、VWMA3 分别代表短、中、长期价格重心。 | 多头时是否 VWMA1 > VWMA2 > VWMA3，空头反过来。 |
| 锁定均线 | 原文希望均线在未继续创新高或新低时保持前值，减少震荡缠绕。 | 这是本策略最需要复核的地方，因为原文没有给完整状态机。 |
| OBV / MAOBV | OBV 用成交量累计价格方向，MAOBV 是 OBV 平滑线。 | 价格突破时，成交量动能是否同向确认。 |
| N 周期突破 | 当前价格突破过去 N 根K线的最高价或最低价。 | 它让策略少在趋势尚未启动时提前入场。 |
| 动态移动止损 | 跟踪开仓后的有利极值，K 随持仓时间从 1 降到 0.3。 | 需要检查它是在保护利润，还是过早截断趋势。 |

## 参数定义

| 参数 | 含义 | v1 默认值 | 来源与说明 |
| :--- | :--- | :---: | :--- |
| M | 基础均线周期 | 20 | 原文只说 M、2M、3M，未给具体值，平台 v1 设为 20。 |
| N | 突破观察周期 | 20 | 原文只说 N 周期高低点，未给具体值，平台 v1 设为 20。 |
| init_stop | 初始止损比例 | 0.02 | 原文未给具体止损幅度，平台 v1 设为 2%。 |
| K_min | 止损系数下限 | 0.3 | 原文明确写到 K 最低 0.3。 |
| decay_step | K 每根Bar衰减步长 | 0.1 | 原文明确写到每过 1 个Bar，K 减少 0.1。 |
| position_size | 成交手数 | 1手 | 原文没有仓位公式，平台 v1 固定 1 手。 |

## 指标计算公式

```python
# VWMA：成交量加权均线。
# 价格先乘以成交量，再除以窗口内成交量总和。
vwma = rolling_sum(close * volume, period) / rolling_sum(volume, period)

# 三条均线构成短、中、长期趋势结构。
vwma1 = VWMA(close, volume, M)
vwma2 = VWMA(close, volume, 2 * M)
vwma3 = VWMA(close, volume, 3 * M)

# 平台 v1 对“锁定均线”的默认复现口径。
# 原文没有给完整状态机，因此当前用多头历史峰值、空头历史谷值近似。
locked_long = cummax(vwma1)
locked_short = cummin(vwma1)

# OBV：上涨日增加成交量，下跌日减少成交量，持平不变。
obv = cumsum(sign(close.diff()) * volume)
maobv = EMA(obv, M)

# N 周期突破线只使用历史窗口，shift(1) 避免用到当前K线数据。
n_high = rolling_max(high, N).shift(1)
n_low = rolling_min(low, N).shift(1)
```

## 开仓逻辑

多头入场需要同时满足：

1. 上一根K线收盘价高于 VWMA3。
2. 上一根K线 MAOBV 高于再上一根K线 MAOBV。
3. 价格高于锁定均线、VWMA2 和 VWMA3。
4. VWMA1 > VWMA2 > VWMA3。
5. VWMA1、VWMA2、VWMA3 的斜率均为正。
6. 当前收盘价突破 N 周期最高价。

空头入场按对称逻辑处理：价格位于均线下方、MAOBV 下降、均线空头排列、斜率为负，并跌破 N 周期最低价。

成交口径：信号在当前K线收盘后确认，实际成交按下一根K线开盘附近执行。当前 v1 固定每次开仓 1 手。

## 出场与风控

策略出场以动态移动止损为核心：

- 多头：跟踪开仓后的最高价，设置向下移动止损。
- 空头：跟踪开仓后的最低价，设置向上移动止损。
- K 值从 1 开始，每持仓 1 根K线减少 0.1，最低降到 0.3。
- 止损幅度 = `init_stop * K`。持仓越久，K 越小，止损线越贴近价格。

多头止损示例：

```text
开仓价 = 480
开仓后最高价 = 500
init_stop = 2%
持仓 5 根K线后，K = max(1 - 0.1 * 5, 0.3) = 0.5
动态止损价 = 500 * (1 - 0.02 * 0.5) = 495
```

这个例子说明：如果价格从 500 回落到 495 附近，策略会认为趋势保护线被触及，于是平仓。

## 平台 v1 默认复现口径

原文有几处没有给出可直接编码的完整公式，平台必须给出默认口径并明确标注：

| 模糊点 | 平台 v1 处理 | 后续复核重点 |
| :--- | :--- | :--- |
| 6个 cond 变量没有逐一定义 | 展开为价格位置、均线排列、均线斜率和锁定均线关系。 | 如果拿到原始公式，应逐项替换。 |
| 锁定均线是否在趋势切换后重置 | 当前用 `cummax/cummin` 近似。 | 这是 v1 最大的复现风险。 |
| 初始止损幅度没有具体数值 | 默认 `init_stop=0.02`。 | 后续要做参数敏感性测试。 |
| 仓位规则没有说明 | 默认固定 1 手。 | 如果进入实盘研究，要改成资金风险口径。 |
| 成交时点没有明确 | 收盘确认信号，下一根K线开盘成交。 | 阶段四要逐笔复核成交时刻。 |

## 研究员需要重点观察

1. 交易是否真的来自“趋势过滤 + 均线形态 + OBV + 突破”的组合，而不是某个单一条件。
2. K线图中 VWMA1、VWMA2、VWMA3 和锁定均线是否能解释入场位置。
3. 动态止损是否过于敏感，导致持仓时间明显短于“中长线趋势跟踪”的定位。
4. 当前品种是否适配该策略。趋势越清晰、成交量配合越明显，策略越可能发挥作用。
5. 后续版本最应该先复核“锁定均线”和“6 个 cond 变量”的精确定义。
