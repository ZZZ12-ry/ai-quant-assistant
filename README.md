# AI 量化助手

AI 量化助手是一个面向期货 CTA 策略研究的本地化工作台。它不是单纯的聊天机器人，而是把策略研究拆成可复现、可检查、可沉淀的五个阶段：策略构思、建模、回测、检查和分析。

当前项目仍处于研究原型阶段，适合用于策略复现、初筛、回测证据整理和研究汇报，不应直接用于实盘决策。

## 核心目标

- 让研究员快速看懂一篇策略文章或交易想法。
- 把阶段一的策略说明转成结构化机器规格。
- 用受控代码生成器生成符合回测引擎接口的策略代码。
- 由固定回测引擎输出交易明细、权益曲线、K线证据和统计指标。
- 在阶段四逐笔复核成交时点、手数、成本、退出原因和极端情况。
- 在阶段五判断策略是否值得继续深入研究，以及下一步优化方向。

## 五阶段流程

### 阶段一：策略构思

输入策略文章、交易想法或研究笔记。平台输出研究员可读的策略说明，并同步生成 `strategy_spec.json` 供阶段二使用。

阶段一强调：

- 核心思想是否讲清楚。
- 关键概念是否解释清楚。
- 入场、出场、手数、风控是否能落地。
- 原文模糊处是否显式标记为平台默认口径。

### 阶段二：建模

阶段二不是让大模型自由发挥，而是把阶段一规格翻译为符合固定接口的 Python 策略代码。

策略代码必须提供：

```text
compute_indicators(df) -> DataFrame
generate_signals(df) -> DataFrame
run(df) -> DataFrame
```

策略层只输出信号列；持仓、成交、手续费、滑点、保证金和交易记录由回测引擎统一处理。

### 阶段三：回测

阶段三由固定回测引擎执行，不由 AI 编造收益、回撤或交易记录。

当前默认口径：

- 日线数据。
- 近五年窗口。
- AKShare / 新浪主力连续合约。
- 单品种验证优先，多品种初筛作为辅助。
- 支持手续费、滑点、保证金、资金不足拒单、涨跌停阻断、低成交量跳过和策略动态止损。

### 阶段四：检查

阶段四定位为交易复核。重点不是泛泛评价策略，而是检查交易是否真的按规则执行：

- 阶段二代码是否忠实复现阶段一。
- 是否存在未来函数。
- 入场、出场、止损、加仓是否和说明一致。
- 成交时点、手数、成本和净盈亏是否可复算。
- 典型交易是否能逐笔解释。

### 阶段五：分析

阶段五只基于真实回测结果和阶段四检查做研究判断：

- 策略是否值得继续深入。
- 收益是否来自稳定规则还是少数行情。
- 风险点在哪里。
- 后续应该先做参数稳健性、多品种验证、数据口径升级还是策略结构调整。

## 样本策略

当前保留一个完整历史样本策略：

- `locked_vwma_obv`：简单有效的均线量化策略。

该样本用于展示完整五阶段结果。当前结论是：它可以作为 v1 基线继续研究，但不能直接推导为可实盘策略。

## 本地运行

安装依赖后启动后端：

```powershell
python -X utf8 -m uvicorn app.main:app --host 127.0.0.1 --port 8015
```

浏览器访问：

```text
http://127.0.0.1:8015/
```

首次使用前可复制配置模板：

```powershell
copy app\config.example.yaml app\config.yaml
```

然后在页面设置 API Key。

## 测试

```powershell
python -X utf8 -m unittest tests.test_platform_contracts
```

## 数据说明

当前默认使用 AKShare / 新浪主力连续合约，适合原型验证和汇报展示。后续若进入更严肃研究，应接入可审计的移仓换月数据，并记录换月日期、展期价差和换月滑点。

## 安全说明

`app/config.yaml`、本地行情数据、回测报告和草稿策略默认不进入 GitHub。不要提交真实 API Key、交易账号、密码或客户数据。
